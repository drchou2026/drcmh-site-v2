這些工具是什麼？

GA4
Google Analytics 4：記錄「誰來看了你的網站」的工具。可以看到每天有多少人造訪、從哪裡來（Google 搜尋 / 社群 / 直接輸入網址）、看了哪些頁面、停留多久。類似診所的「訪客簽到簿」，但是數位版、自動記錄。
sitemap.xml
網站地圖：一份給 Google 機器人看的「目錄清單」，列出網站所有頁面的網址。Google 的爬蟲會先讀這份清單，再依序去每頁收錄。有 sitemap，Google 就不容易漏掉你的新文章。本站已自動生成，無需手動維護。
robots.txt
爬蟲規則檔：告訴 Google「哪些頁面可以收錄，哪些不行」的設定檔。例如後台管理介面不應該被 Google 索引，robots.txt 就負責把它擋掉。本站已設定後台 /keystatic 排除在外。
OG Image
社群分享預覽圖：當網址被貼到 LINE、Facebook、Twitter 時，自動出現的那張大圖。本站每個主要頁面都已設定專屬 OG Image（1200×630 px），分享時不會只顯示空白或小圖。封面圖存放於 /public/images/og-*.jpg。

結構化資料 JSON-LD

結構化資料是「寫給 Google 和 AI 搜尋引擎看的補充說明」。把診所資訊以機器讀得懂的格式寫進網頁 code 裡，讓 Google、ChatGPT 等 AI 在回答搜尋者問題時，能更正確地呈現你的地址、電話、專科、營業時間等資訊。

→Physician — 醫師資歷、專長、執照認證
→FAQPage — 首頁 FAQ 觸發 Google 折疊式富有結果
→Article / MedicalWebPage — 每篇衛教文章頁面
→BreadcrumbList — 網站導航結構
→LocalBusiness — 在地商家搜尋補充訊號

08

如何與AI 協作來改網站

本專案採用 AI 輔助開發流程，未來功能迭代可透過以下方式高效推進

名詞解釋

GitHub
存放程式碼的地方，也負責版本控制。類似「程式碼專用的雲端硬碟」，每一次修改都有歷史紀錄可追溯，不怕改壞找不回來。
IDE 是什麼
IDE（整合開發環境）就是「改程式碼的地方」，最常見的是 VS Code。可以想成 Word 之於文字、VS Code 之於程式碼。
AI 在哪？
AI 工具可以「插件」形式嵌入 VS Code，讓你直接在開發環境裡使用：
  • Claude Code（Anthropic）— 本專案主要工具，CLI 形式操作整個 codebase
  • GitHub Copilot（Microsoft）— 直接內建在 VS Code 中，即時補全程式碼
  • Codex / Cursor / Windsurf 等 — 同樣是 IDE 型 AI 輔助工具
⚠️ 以上工具皆需訂閱付費才能使用完整功能
AI 模式
在 IDE 中使用 AI 通常有三種模式：
  • Edit 模式：AI 直接修改你指定的程式碼
  • Ask 模式：問 AI 問題、請它解釋程式碼，不會直接動檔案
  • Plan 模式：請 AI 先規劃實作步驟再執行，適合較大的功能開發
開發工具鏈
VS Code + Copilot / Claude Code / Codex
AI 插件嵌入 IDE，即時程式碼補全與修改。降低維護門檻，讓新接手的開發者快速上手。
Git + GitHub
版本控制核心。所有變更皆有歷史紀錄可追溯，Push 觸發自動部署，Pull Request 支援多人協作審查。
Keystatic Cloud（後台）
醫師／助理可直接在瀏覽器操作後台更新內容，完全不需接觸程式碼或開發工具。

標準協作流程

1
需求描述
醫師／助理以自然語言描述需求（如：「我想在首頁加一個停車資訊區塊」）
2
AI 輔助開發
開發者透過 Claude Code 快速完成實作，AI 負責撰寫程式碼，開發者確認與審查
3
GitHub 提交與自動部署
確認後合併至 main 分支，Cloudflare Pages 自動觸發部署，約 2 分鐘上線
4
日常內容維護（無需開發）
醫師／助理登入 drcmh.com/admin，透過視覺化後台更新文章、消息、時刻表，不需接觸任何程式碼
AI 撰文協作建議：醫師提供衛教文章的醫學大綱（症狀、成因、治療選項），交由 AI（ChatGPT / Claude）撰寫初稿，醫師審閱修正後發布。提示詞建議加上：「台灣醫療情境、繁體中文、面向一般病患的白話文、字數 1500 字以上」。
⚖️ 台灣醫療法合規提醒

無論是 AI 生成的網站內容、衛教文章或社群貼文，醫師在發布前必須親自審閱，確保符合台灣相關法規。常見應注意事項：

  • 醫療法第 85 條：醫療廣告不得宣稱療效保證、比較競爭對手、誇大不實
  • 醫療法第 86 條：不得以優惠、折扣、贈品等方式招徠病患
  • 藥事法：不得對藥品、醫療器材進行不實廣告
  • 個資法：病患個人資料（含照片、案例）不得未經同意公開使用

→ 若有疑慮，請諮詢醫師公會或法律顧問後再發布。AI 不能取代醫師的專業判斷與法律責任。
